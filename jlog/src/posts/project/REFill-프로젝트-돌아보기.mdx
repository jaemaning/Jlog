## 🧬 기술 스택
TypeScript, React, Redux, styled Components, Material-UI

## 🌈 위 기술을 선정한 이유는?

우선 비대면 탈모 상담을 진행하게 될 때 Mobile App 과 Web 중 고민을 했었는데, 아무래도 진지한 상담을 진행하게 될 것이라 생각하였고, 병원 근무중인 환경에선 인터넷 통신 상태가 상대적으로 양호한 Web 으로 진행하는 것이 더 효과적이라고 생각하였습니다.


> 1. TypeScript : TypeScript를 선정한 이유는 사실 TypeScript를 선정하지 않았을 때 리스크가 너무 크기 때문이었습니다. JavaScript로 프로젝트를 구성하게된다면 생산성은 올라가겠지만 어느 순간 발생할지도 모르는 type 에러로 인해 디버깅에 너무 많은 리소스가 낭비될 것을 우려해 보험을 든다는 생각으로 TypeScript를 선정하였습니다.
> 2. React : 3명의 FrontEnd 의 현재 사용 가능한 기술 스택 정도와 3주라는 짧은 개발 시간을 고려할 때 Reference 가 많고 러닝커브가 낮은 React와 Vue 중 고민을 했고 최종적으로 자유도가 높아 원하는대로 사용하기 편리한 React를 선택하게 되었습니다.
> 3. Redux : React 를 선택하게되면 자연스럽게 따라오는 상태관리 라이브러리는 최근 성능적으로 향상되고 빠른 Recoil 같은 라이브러리들도 있지만 저희는 상태 관리를 해야할 데이터가 많지 않았기 때문에, 기존에 사용해본 Redux 를 사용하기로 했습니다.
> 4. Styled Components, Material UI : UI 를 구성할 때 도움을 주는 라이브러리들이 상당히 많지만, 저희는 자유도를 높이되 최대한 빠른속도로 개발을 해야 했으므로, MUI 에서 제공하는 편리함과 Styled Components 가 주는 자유도를 적절이 섞어 개발에 임했습니다.

## 📌 프로젝트를 통해 얻은 것들.
첫 번째 팀 프로젝트로 잘 모르는 것 투성이였던 6명이 힘을 합쳐 프로젝트를 완성 했다는 경험이 가장 큰 수확인 것 같습니다. 처음엔 불가능해 보였던 프로젝트도 다 같이 힘을 합쳐 진행하니 힘든 시간도 많았지만 결국 해낼 수 있었고 할 수 있다는 자신감을 얻었습니다.

> 1. 동기와 비동기의 차이를 구분하고 web 에서 코드들이 동작하는 방식을 알게 되었습니다.
> 2. React 의 State 와 LifeCycle, Rendering 등 필수적으로 알아야할 React 개념에 대해 다시 한번 학습하는 기회가 되었습니다.
> 3. openVidu API, kakao map API 등 외부 API 를 활용하는 방법에 대해서 알게 되었습니다.
> 4. Git, Jira 등 버전관리 및 프로젝트 Managing 툴을 사용하고 익히며 팀으로 진행하는 프로젝트에서 협업하는 방식을 배웠습니다.
